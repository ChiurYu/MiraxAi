# Mirax AI 人工提示词

本文用于在新的 Codex 或 Claude Code 任务中恢复 Mirax AI 工作流。

默认方式是 **Codex 直接接管规划、实现、验证和进度同步**。只有用户明确要求「交给 Claude Code」「用 cmux 派工」或「只负责下任务和验收」时，才使用后面的委派模板。

## 固定入口

- 项目目录：`/Users/yuzhenzhao/code/ai/Mirax AI`
- 活动状态：`docs/codex/PROJECT-STATE.md`
- 后续新计划：`docs/codex/plans/`
- 历史计划与设计：`docs/superpowers/plans/`、`docs/superpowers/specs/`
- 人工截图输入：`docs/截图/`
- 临时调度状态：`.codex/dispatch-state.json`，它不是项目进度源

恢复顺序：`AGENTS.md` → `CLAUDE.md` → `docs/codex/PROJECT-STATE.md` → 状态文件明确指向的当前计划。

## Codex 继续完成工作

日常推荐把下面代码块直接发给 Codex：

```text
继续完成 Mirax AI 当前工作。

工作目录：
/Users/yuzhenzhao/code/ai/Mirax AI

请先读取并遵守：
1. AGENTS.md
2. CLAUDE.md
3. docs/codex/PROJECT-STATE.md
4. PROJECT-STATE.md 明确指向的当前计划和相关设计、证据、测试

执行要求：
- 先运行 git status --short，保留并避开已有无关改动。
- 从 PROJECT-STATE.md 判断当前任务，不按目录日期猜测计划。
- 默认直接实现下一个明确任务，并运行计划要求的验证。
- 不把 mock、未连接或未完成真实 dogfood 的能力写成已完成。
- API Key、token、完整本地路径和临时 OSS URL 不得进入普通 snapshot、日志或错误信息。
- 只有验收证据明确时才更新 legacy-ui-gap-list.md 状态列。
- 完成后同步当前计划与 docs/codex/PROJECT-STATE.md，明确已完成、当前阶段、下一步和仍需规划。
- 不要修改、暂存或提交与当前任务无关的工作区改动。

最后汇报：
- 结果
- 修改文件
- 验证命令与结果
- 项目状态更新
- 阻塞或仍待人工验证的事项
```

## Codex Review / 验收

```text
请只 review 和验收 Mirax AI 当前改动，不继续实现或扩大范围。

工作目录：
/Users/yuzhenzhao/code/ai/Mirax AI

请读取：
- AGENTS.md
- CLAUDE.md
- docs/codex/PROJECT-STATE.md
- 当前计划

请检查：
1. git diff 是否只包含当前任务允许的文件。
2. 实现是否满足当前计划和安全边界。
3. 计划要求的测试、typecheck、build 或人工验证是否有新鲜证据。
4. 是否把 mock、not-connected 或未完成真实 dogfood 的能力误写为成功。
5. 是否可以更新当前计划、PROJECT-STATE.md 或 legacy-ui-gap-list.md。

先按严重程度列出问题；没有阻塞问题时明确说明。最后给出：通过、需要返工或阻塞。不要自动 push。
```

## Claude Code 显式委派备用

只有用户明确要求交给 Claude Code 时才使用：

```text
你是 Mirax AI 的 Claude Code 实现工位。请严格遵守仓库 CLAUDE.md。

工作目录：
/Users/yuzhenzhao/code/ai/Mirax AI

请先读取：
1. AGENTS.md
2. CLAUDE.md
3. docs/codex/PROJECT-STATE.md
4. PROJECT-STATE.md 明确指向的当前计划
5. 当前任务相关源码、测试、设计与证据

只完成当前计划中的下一个明确任务：
- 先运行 git status --short，避开已有无关改动。
- 只修改任务允许的文件；不跨阶段继续做下一项。
- 不修改 .codex/dispatch-state.json。
- 不修改 legacy-ui-gap-list.md，除非计划明确要求且有验收证据。
- 遇到证据不足、接口语义不清或产品行为不确定时返回 BLOCKED，不要脑补。
- 运行当前任务要求的验证，但不要 commit、不要 push。
- 默认不修改计划 checkbox 或 docs/codex/PROJECT-STATE.md；把建议的状态更新写在报告里，由 Codex review 后同步。

按以下格式报告：
STATUS: DONE | BLOCKED | IN_PROGRESS
CURRENT TASK:
CHANGED FILES:
VERIFICATION:
SUGGESTED STATE UPDATE:
NOTES:
```

## cmux 终端 Claude Code 委派备用

只有用户明确要求 cmux 派工时才使用：

```text
请作为 Mirax AI Codex 总控，通过 cmux 的普通 terminal Claude Code 工位派发当前任务；不要自己实现 worker 任务。

工作目录：
/Users/yuzhenzhao/code/ai/Mirax AI

请先读取：
- AGENTS.md
- CLAUDE.md
- docs/codex/PROJECT-STATE.md
- PROJECT-STATE.md 明确指向的当前计划
- 当前可用的 cmux 与 cmux-workspace skill

要求：
1. 根据 PROJECT-STATE.md 确定任务，不按文件名日期猜测。
2. 优先复用当前 Mirax AI workspace；只有任务独立或冲突时才新建 workspace。
3. 必须使用普通 terminal 中的 Claude Code，不得创建或使用 Claude Code · React agent-session。
4. 用 list-pane-surfaces 和 read-screen 验证能看到 Claude Code、kimi-for-coding 和正确 cwd 后才能派工。
5. 通过 terminal surface 的 send + Enter 派发完整 worker 提示词，并用 read-screen 确认已收到。
6. 只有真实派工成功后才写 dispatch_route=terminal.claude 和 worker_surface_ref。
7. worker 不得 commit、push 或修改无关文件。
8. 只有用户要求持续监控时才创建 heartbeat；使用产品提供的 automation 工具，不手写自动化文件。

派工成功后只汇报 task、workspace_ref、worker_surface_ref、dispatch_route 和验证画面，不继续实现。
```

```text
请汇报 Mirax AI 当前任务进度，不修改文件。

读取 AGENTS.md、CLAUDE.md、docs/codex/PROJECT-STATE.md 和它明确指向的当前计划，然后运行 git status --short。

请给出：
1. 当前产品阶段和当前任务。
2. 当前计划路径。
3. 已完成项和第一个未完成项。
4. 哪些能力仍是 mock、未连接或待真实 dogfood。
5. 当前未提交改动及其是否属于当前任务。
6. 下一步由 Codex 直接做，还是需要用户明确授权后委派 Claude/cmux。
```

## 终端查看进度

查看项目状态：

```bash
sed -n '1,240p' docs/codex/PROJECT-STATE.md
```

查看当前计划 checkbox：

```bash
PLAN="$(rg -o 'docs/(codex|superpowers)/plans/[0-9]{4}-[0-9]{2}-[0-9]{2}-[^` ]+\.md' docs/codex/PROJECT-STATE.md | tail -1)"
[ -n "$PLAN" ] && rg -n "^- \[ \]|^- \[x\]" "$PLAN"
```

查看工作区与临时调度状态：

```bash
git status --short
sed -n '1,220p' .codex/dispatch-state.json
```

一句话继续：

```text
读取 AGENTS.md、CLAUDE.md 和 docs/codex/PROJECT-STATE.md，由 Codex 直接完成当前下一项并验证；保留已有无关改动，完成后同步项目状态。
```
