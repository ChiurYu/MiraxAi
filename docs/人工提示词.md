# Mirax AI 人工提示词

本文用于在新开的 Codex / Claude Code 窗口中手动复制提示词，恢复 Mirax AI 当前工作流。

当前优先推荐：**用户直接把实现任务安排给 Claude Code；Claude Code 完成后，用户再通知 Codex 做 review / 验收。**

Codex 后续默认不再自动派工，也不主动创建 cmux worker。只有用户明确说「自动调度 / 用 cmux 派工 / 开始干活」时，才恢复 `cmux-agent-dispatch` 流程。

## 当前协作方式

1. 用户把任务直接发给 Claude Code。
2. Claude Code 负责实现、运行验证命令，并报告改动文件和验证结果。
3. Claude Code 不要 commit / push，除非用户明确要求。
4. Claude Code 做完后，用户告诉 Codex：「Claude 做完了，请 review」。
5. Codex 只做 review / 验收：检查 diff、运行必要验证、指出问题、必要时给 Claude 修改提示词。
6. 验收通过后，是否提交、推送或继续下一步，由用户再决定。

复制规则：

- 只复制代码块里的内容，不复制标题和说明文字。
- 每个提示词代码块已经去掉起止标记，直接复制完整代码块即可。
- 尖括号里的内容（例如 `<当前任务相关文件>`）需要替换成真实任务信息。

## 和 Claude Code 沟通的基本原则

- 每次只给 Claude 一个明确任务，不要一次塞多个阶段。
- 明确允许修改和禁止修改的文件范围。
- 明确验证命令，让 Claude 自己先跑。
- 要求 Claude 完成后按固定格式汇报：`STATUS`、`CHANGED FILES`、`VERIFICATION`、`NOTES`。
- 让 Claude 把不确定的地方写成「待确认问题」，不要让它脑补旧版行为。
- 实现任务里默认禁止 Claude commit / push。
- 如果 Claude 做偏了，直接发「停止当前方向，只按下面范围修正」并重申允许修改文件。

## Claude Code 继续实现短提示词

日常推荐优先复制这一段。它会让 Claude Code 自己读取规划文档、检查当前进度，并从第一个未完成任务继续。

```text
你是 Mirax AI 的 Claude Code 实现工位。请严格遵守仓库 CLAUDE.md。不要 commit，不要 push。

工作目录：
/Users/yuzhenzhao/code/ai/Mirax AI

请先读取并自行判断当前进度：
1. AGENTS.md
2. CLAUDE.md
3. docs/superpowers/PROJECT-STATE.md
4. PROJECT-STATE.md 中指向的当前计划 / 下一步文档
5. 和当前任务相关的 spec、页面卡、功能卡、测试或源码文件

请运行：
- git status --short
- 从当前计划中找第一个未完成的 `- [ ]` 任务；如果计划已完成，就按 PROJECT-STATE.md 的「下一步」继续。

执行要求：
- 只做下一个明确任务，不要跨阶段连续做多个任务。
- 优先遵守计划中的文件范围、验收标准和验证命令。
- 如果计划没有写清楚文件范围，请先根据代码结构判断最小修改范围，并在 NOTES 里说明。
- 遇到不确定的产品行为、旧版证据不足、接口语义不清或数据来源不明时，写入「待确认问题」，不要自行脑补。
- 不要修改 `.codex/dispatch-state.json`。
- 不要修改 `docs/reverse-engineering/legacy-ui-gap-list.md`，除非计划明确要求。

完成后先运行相关验证命令，然后按以下格式报告：

STATUS: DONE | BLOCKED | IN_PROGRESS
CURRENT TASK:
- <你判断并执行的是哪个计划 / 任务 / 文件>
CHANGED FILES:
- ...
VERIFICATION:
- <命令> → <结果>
NOTES:
- <重要说明、风险、待确认问题>
```

```text

2. 修改是否只包含任务允许范围。
```

## Codex 总控自动调度提示词（备用）

只有在需要 Codex 自动派工给 cmux worker 时，才把下面这段复制到新开的 Codex 窗口：

```text

你现在是 Mirax AI 项目的 Codex 总控，工作目录是：
/Users/yuzhenzhao/code/ai/Mirax AI

请使用 cmux-agent-dispatch 流程继续当前任务，但先不要假设旧的 dispatch-state 一定可信。

请按顺序执行：

1. 读取并遵守：
   - AGENTS.md
   - CLAUDE.md
   - docs/superpowers/PROJECT-STATE.md
   - PROJECT-STATE.md 中「当前自动调度入口」指向的计划文件
   - /Users/yuzhenzhao/.codex/skills/cmux-agent-dispatch/SKILL.md
   - /Users/yuzhenzhao/.codex/skills/cmux/SKILL.md
   - /Users/yuzhenzhao/.codex/skills/cmux-workspace/SKILL.md

2. 重新执行 Phase 0：
   - 按 AGENTS.md 规则，从 docs/superpowers/plans/ 里按文件名日期选择最新计划。
   - 列出计划中所有未完成的 `- [ ]` 步骤。
   - 判断每个步骤归属：
     - worker：写页面卡、写功能卡、代码实现、测试、浏览器验证。
     - controller：验收、提交、勾选计划、更新 dispatch-state、创建 heartbeat。
   - 重新写入 `.codex/dispatch-state.json`。
   - 不要直接复用旧 dispatch-state 里的 current_task。

3. 如果当前任务需要 worker，执行 Phase 1.5 cmux discovery：
   - 优先复用当前 Mirax AI workspace；同一主题 / 同一任务链不要随便新开 workspace。
   - 需要强相关并行任务时，在同一个 workspace 内分屏或增加 surface。
   - 只有任务独立、容易冲突，或当前 workspace 无法承载时，才新建 workspace。
   - 所有 cmux 命令先设置：
     `CMUX="$(command -v cmux || true)"; [ -n "$CMUX" ] || CMUX="/Applications/cmux.app/Contents/Resources/bin/cmux"`
   - 使用 `identify --json --id-format both`、`list-workspaces --json --id-format both` 获取 workspace_ref 和 workspace_id。
   - Mirax AI 默认使用普通 terminal surface 里的 Claude Code，不要创建 `--type agent-session --provider claude`。
   - 如果需要新 cmux 窗口，使用：
     `WIN="$("$CMUX" new-window | awk '/^OK / {print $2}')"`
     然后：
     `"$CMUX" new-workspace --window "$WIN" --name "Mirax AI Claude" --cwd "/Users/yuzhenzhao/code/ai/Mirax AI" --command "claude" --focus true`
   - 如果在已有 workspace 增加工位，创建 terminal surface 后运行：
     `"$CMUX" send --workspace <workspace_ref> --surface <surface_ref> 'cd "/Users/yuzhenzhao/code/ai/Mirax AI" && claude'`
     再发送 Enter。
   - 必须用 `list-pane-surfaces` 和 `read-screen --lines 40` 验证画面是普通 Claude Code 终端，能看到 `Claude Code v...`、`kimi-for-coding` 和项目 cwd。
   - 如果看到 `Claude Code · React`，说明开错了 agent-session，停止并重建 terminal Claude Code 工位。

4. 执行 Phase 2 派工：
   - 通过已验证的 terminal Claude Code surface 派工：
     `"$CMUX" send --workspace <workspace_ref> --surface <worker_surface_ref> "<worker prompt>"`
     然后：
     `"$CMUX" send-key --workspace <workspace_ref> --surface <worker_surface_ref> Enter`
   - 用 `read-screen` 确认 Claude 已收到任务或开始处理。
   - 只有确认成功后，才把 current_task.status 改成 `dispatched`，并把 `dispatch_route` 写成 `terminal.claude`。
   - 不要使用 `workspace.prompt_submit`，不要把 React agent-session 当成 worker。

5. 创建 / 更新 heartbeat：
   - 必须先用 tool_search 找 automation_update。
   - 不要手写 `~/.codex/automations/` 文件。
   - heartbeat 检查 worker 完成时，`dispatch_route=terminal.claude` 使用 `cmux read-screen` + `git diff` 判断。

6. 派工成功后停止当前回合：
   - 汇报 task id、workspace_ref、worker_surface_ref、dispatch_route、window_ref（如果有）。
   - 不要自己实现 worker 任务。

注意：
- 当前阶段优先使用 `docs/截图/` 中的人工截图，不要重复操作旧版 App 截图。
- 不要更新 `docs/reverse-engineering/legacy-ui-gap-list.md` 状态列，除非计划明确要求。
- 不要修改计划范围外文件。
- 不要 commit/push，除非你是 controller 验收通过后按计划提交。

```
