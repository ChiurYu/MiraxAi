## 用户目标

用户希望从对标素材或输入内容出发，在首页工作台按编号步骤完成文案、声音、数字人、成片和发布准备的完整链路。

## 页面结构

| 区域 | 控件或字段 | 行为 |
| --- | --- | --- |
| 顶部模式切换 | 手动 / 自动 / 草稿 | 切换工作模式，影响主工作区展示 |
| 左侧导航 | 首页、形象管理、素材管理、创作管理、任务中心、账号管理、设置、帮助 | 全局导航入口 |
| 主工作区 | 7 个编号卡片：学习对标、改写文章、声音生成、视频生成、一键成片、标题封面、视频发布 | 每个卡片包含输入字段、选择器和生成按钮 |
| 右侧预览 | 视频预览区 | 展示生成结果或占位提示 |

## 输入与输出

| 类型 | 说明 |
| --- | --- |
| 用户输入 | 对标视频链接、改写提示词、字数、声音样本、数字人形象、字幕/BGM/音量等成片参数 |
| 系统输出 | 提取文案、改写文案、合成语音、数字人视频、完整成片、标题/封面/话题/标签 |
| 下游依赖 | 发布流程（FC-PUBLISH-PREP）、任务中心（EV-RUNTIME-100）、声音/形象/素材管理（FC-ASSET-MANAGEMENT） |

## 执行链路

| 层级 | 证据或预期职责 |
| --- | --- |
| 前端 | 首页工作台展示 7 步流程卡片，收集用户输入，显示生成按钮和状态 |
| Provider | AI Provider 负责文案提取、改写、声音克隆、语音合成、数字人视频生成；Publish Provider 负责发布前准备 |
| Sidecar | FFmpeg 用于视频/音频合成；Python 本地服务用于视频解析、语音/数字人模型调用 |
| 本地存储 | SQLite 保存草稿、任务、素材、账号、配置；localStorage 临时保存未提交的草稿字段 |
| 外部服务 | 云端 API 用于账号认证、激活校验、平台 AI 配置拉取 |

## 限制与风险

| 风险 | 影响 | 处理方式 |
| --- | --- | --- |
| 必须登录账号 | 未登录时无法进入首页工作台 | Mirax AI 第一版先做本地 mock 流程，不强制云端登录 |
| 必须激活会员 | 未激活时弹出激活码弹窗，限制完整功能 | Mirax AI 第一版不做激活码限制，仅做本地依赖检查 |
| 依赖云端 AI 配置 | 旧版启动时拉取云端 AI 配置，失败时功能受限 | Mirax AI 改为用户自行配置 Provider 密钥和 Base URL |
| 真实生成需要本地模型/服务 | 声音克隆、数字人、渲染依赖本地 Python/FFmpeg | Mirax AI 通过 sidecar-manager 抽象本地依赖检查 |

## Mirax AI 实现建议

决策：重做。

理由：复刻旧版 7 步生产能力和流程顺序，但使用 Mirax AI 的新版信息架构与视觉设计。保留用户自行配置 Provider、本地 mock 优先、不强制登录/激活的桌面端体验。
